Description of Capabilities

This release includes the following components:

 

·        Questra Service Agent 5.2 with patch 99.

·        ContactGE, ConnectToGE, File Transfer, Remote Service Mode and Multi Tunnel.

·        Ability to host web applications using Cookie Monster.

·        Common Service Desktop supported on IE6.0 and Mozilla Firefox 5.0.

·        Agent Configuration Tool for 5.2 Agent.

 

This release package includes a more detailed integration guide for the components above.

Pre-Installation Procedure

Before installing InSite2.0 platform (Device Side), please make sure that:

 

1.      Previous releases of InSite2.0 Agent and Cookie Monster are fully uninstalled from the machine (For un-installation, please refer Un-installation Procedure section of this note).

 

2.      Ports 8081, 8020, 8011, 80, 8080, 8005 and 8009 should be free on machine, also make sure that VNC and Telnet should be available at the needed ports.

 

 

An InSite 2.0 platform depends on the following software packages. These packages need to be installed in the system prior to installing the Insite2.0 platform. Please go through the below details before trying to install Insite2.0 platform.

1.       Java: InSite 2.0 requires JRE Version 1.5.x. This can be downloaded from java.sun.com site. Modalities shall set JAVA_HOME variable pointing to home directory of Java. Please copy tools.jar to %JAVA_HOME%\lib directory. Tools. jar is not part of JRE, It is  available in JDK package in its lib directory.

Note: If you have JAVA_HOME pointing to JRE 1.4 then please create an environment variable CKM_JAVA_HOME and point it to JRE 1.5. Also, copy tools.jar to %CKM_JAVA_HOME%\lib directory

 

2.       Perl: Insite2.0 requires a latest version of Perl to be installed in the system. Its highly recommends that you download the Perl from the below link and install the same prior to installing the Insite2.0 platform. If you install any other version of Perl, due to non-compatibility, your Insite2 platform may not function properly.

 

NOTE: Perl is only required for windows. Linux already comes with Perl5.

 

      http://download.openge.ge.com:81/insite2/Perl.ZIP

 

Steps to install the Perl package:

·        Download the Perl.zip file from the above-mentioned link.

·        Extract the zip file to a temporary folder.

·        Execute the perl.bat file with the below syntax.

Ex. perl.bat install <targetDir>

·        After installing perl, close the command window where you installed the perl package. All environment variables will set as system variable

·        Take new command window for next steps.

 

Installation Procedure

Windows:

Steps to install InSite2.0 platform:

 

1.       Request the Insite2.0 Release Package from GST.

2.       Extract the package into a temporary folder.

3.       Edit the platform.xml file to change installation folders of your choice so that the CKM and QSA will get installed in the folders of your choice.

4.       Save the platform.xml file with your changes and quit.

5.      Configure the GenericInstaller.xml file as per your device type configuration.

6.       Open a new command window and change the directory to the temporary folder where you have extracted the Insite2.0 package.

7.       Execute perl platform.pl command to install the Insite2.0 platform (CKM+QSA). It is assumed that you have installed Java1.5 and Perl as mentioned above in this release notes in the Internal dependencies section and updated the PATH system environment variable to have the perl path prior to installing Insite2.0.

8.      Close the command window after successful installation.

 

Installation Procedure of Common Service Desktop:

 

·        Extract modality-csd.war into a directory called modality-csd.

Note:   We recommend using ANT, JAR or PKZIP to extract CSD.

If you are using WinZip to extract CSD, please go to View Menu of WinZip and select "Files by Folder (Explorer Style)". Then you do Ctrl+A to extract all the files and folders and extract to the modality-csd directory.

 

·        Modalities need to add the below lines to the httpd.conf file to enable the csd using Insite2.0 platform.  Add below lines to httpd.conf (located in the Apache/conf directory of CookieMonster) file.

Alias /CSD_ALIAS/WHERE_IMAGES_ARE/ "C:/CKM/modality-csd/<images-dir>/" [use lowResImages directory for low Resolution and images directory for high Resolution. Move index.html.low to index.html for low Resolution]. The alias directory depends and changes on where the modality-csd is extracted.

Note: The above location "C:/CKM/modality-csd/<images-dir>/" is just taken as an example. Please locate your own path of modality-csd.

 

Installation of Config Tool:

 

1.       Please reboot the machine in order to use the configuration tool.

2.       Extract InSite2ConfigTool.war to a directory called InSite2ConfigTool.

3.       Copy modality specific Config_Tool.html to InSite2ConfigTool directory instead of default that is present in InSite2ConfigTool.war. Config_Tool.html provides a parameter, which the modality can change. Below is the parameter example:

<param name="userRole" value=Customer>

The Value of the parameter name "userRole" can be changed to either Customer or ServiceEngineer.

4.       Run config.pl with -e option with the config.xml (i.e., perl config.pl e config.xml) [both are present under %WIP_HOME% dir] in the Cookie Monster to install CSD. Make sure InSite2ConfigTool and modality-csd path are put correctly in the config.xml file.

5.       Please go through the following steps to configure InSite2 Config Tool:

·        After the agent is installed, go to the bin directory of agent. The agent installation directory is specified in Step 4 of Installation Procedure of Insite2 platform.

·        Run gencfg command as specified

                gencfg -template ..\etc\templates\qsa -cfgdir ..\etc

 After executing this command you can use Config Tool to configure the Agent. There is no change in the use of Config Tool from the previous release.

6.       Start Cookie Monster

7.       Invoke http://localhost/modality-csd/ in browser

8.      Click on Configuration Tab -> you will see Insite2 Config Tool. Click on it. You will get Configuration application on Right Frame.

Linux:

Steps to install InSite2.0 platform:

 

1.       Request the Insite2.0 Release Package from GST.

2.       Extract the package into a temporary folder.

3.       Execute the command: chmod +x *

4.       Execute perl install.pl command to install CookieMonster in the current directory.

5.       CSD and InSite2.0 Config Tool automatically get installed under CookieMonster/tomcat/webapps directory.

6.       To install Agent, Execute the command: ./Insite2setup.sh

 

Start and Stop Procedure

Windows:

To start/stop/probe CookieMonster use the below commands respectively.

 

·        Open command prompt and change directory to CookieMonster install path.

·        Execute CookieMonster start to start.

·        Execute CookieMonster stop to stop.

·        Execute CookieMonster probe to probe.

 

Start Procedure for Agent:

 

·        Open command prompt and change dir to Questra install directory and to the bin dir from there.

·        Execute qsamain menu config ../etc/qsaconfig.xml

·        To stop Questra agent, follow the menu/instructions on the console where you have started the agent (qsamain).

 

Linux:

To start/stop CookieMonster use the below commands respectively.

 

·        Change the permission as 777 of <CookieMonster installation directory>/tomcat/logs/catalina.out file.

·        Change to root user and Execute ./CookieMonster.sh start to start.

·        Execute command  - chmod +x tomcat/webapps/modality-csd/* (at the installation time only)

·        Execute command  - ln -sf <CookieMonster installation directory>/Apache/lib/libapr-0.so.0 /usr/lib/libapr-0.so.0

·        Execute command  - ln -sf <CookieMonster installation directory>/Apache/lib/libaprutil-0.so.0 /usr/lib/libaprutil-0.so.0

·        Execute ./CookieMonster.sh stop to stop.

 

Start Procedure for Agent:

 

·        Append the lib directory of InSite2 Agent(../dsp/agent/lib) to LD_LIBRARY_PATH environment variable.

·        Go to bin directory of Agent and Execute the Command : gencfg -template ../etc/templates/qsa -cfgdir ../etc.

·        Make sure CookieMonster is running. Invoke http://localhost/modality-csd/ in browser.

·        From the Agents bin directory only, Execute qsamain menu config ../etc/qsaconfig.xml

·        To stop Questra agent, follow the menu/instructions on the console where you have started the agent (qsamain).

 

Note: Since we recommend JDK 1.5, it may happen that the Mozilla browser (default version 1.4) will not display any tabs/button/text-boxes on the Config_Tool.html page shown on CSD browser. In that case, use the following command to create the symbolic link for Mozilla plug-in:-

Command: ln -s JAVA_HOME/jre/plugin/i386/ns7/libjavaplugin_oji.so /usr/lib/mozilla-1.7.10/plugins/

Un-Installation Procedure

Windows:

Step to uninstall InSite2.0 Platform:

·        To uninstall the Insite2.0, open a command window and change directory to the folder where you installed the CookieMonster (Ex. C:\CKM).

·        Execute the command perl platform.pl a uninstall

·        The above command will uninstall Insite2.0 platform (CKM+QSA).

 

Linux:

Step to uninstall InSite2.0 Platform:

·        To uninstall the Insite2.0 Agent, change to the directory where you installed Agent.

·        Execute the command: _uninst\uninstaller.bin.

·        To uninstall CookieMonster, delete CookieMonster Directory. There is no environment variable dependency for CookieMonster.

·        The above commands will uninstall Insite2.0 platform (CKM+QSA).

FAQ

1.  What should I do if I get an error message  The system cannot find the path specified? I cannot see cache sub-directory in the modality-csd directory where the files are being looked for?

Ans:  This happens if you use WinZip and press Ctrl+A to extract all the files. In WinZip application, go to View Menu and select "Files by Folder (Explorer Style)". Then you do Ctrl+A to extract all the files and folders.

 

2.  What can be the issue if I get an error message thrown by the Java Applet stating %INSITE2_ROOT_DIR%\etc\<XML File Name> couldn't be found while navigating to ConfigTool from http://localhost/modality-csd/. ?

Ans: Restart the system after JAVA 1.5 installation. This generally happens in case Java is not installed on the machine where agent installation is to be performed and we do not restart machine after Java installation.
